__1\. 引言 \(Introduction\)__

__1\.1 编写目的：__ 本文档旨在明确“基于 LLM 与智能代理的独立游戏资产自动化构建系统”的功能与非功能需求，为后续的系统设计、编码实现及系统测试提供学术及工程依据。

__1\.2 项目定位：__ 本项目是一个运行在本地 VS Code 工作区内的开发者辅助工具（Developer Tool）。通过自然语言指令，自动化生成游戏项目所需的结构化配置文件（JSON/CSV）及相关素材的提示词文案。。

__2\. __核心需求 \(Functional Requirements\)

· __FR\-01 命令行交互与指令解析：__ 系统通过 Cline 在 VS Code 终端接收用户的自然语言需求（如：“生成一个带有3种技能的Boss配置文件”）。

· __FR\-02 结构化数据生成：__ 系统通过调用 DeepSeek API，将非结构化文本转换为严格符合游戏设定的 JSON 数据（包含生命值、攻击力、技能描述等）。

· __FR\-03 本地文件系统自动化 \(I/O\)：__ 系统需利用 Cline 的文件读写权限，自动在当前 VS Code 工作区的指定目录（如 \./assets/configs/）下创建、覆盖或更新 \.json 文件。

· __FR\-04 文本类美术提示词生成：__ 系统需根据生成的游戏实体属性，自动推导并生成用于后期 AI 绘画的标准化 Prompt（纯文本），并保存为对应的 \.txt 或 \.md 描述文件。

 __3\. __非功能需求 \(Non\-Functional Requirements\)

· __NFR\-01 数据容错性：__ 必须具备数据格式校验机制。若 DeepSeek 返回的 JSON 格式损坏或缺少必填字段，脚本需具备自动拦截报错或请求大模型重试的能力。

· __NFR\-02 轻量化运行：__ 不依赖庞大的前端框架或本地 GPU 算力，所有网络请求仅限于与 DeepSeek API 的轻量级文本通信。
