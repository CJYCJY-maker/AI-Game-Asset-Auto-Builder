### 独立游戏资产与配置自动构建器 \(Indie Game Asset & Config Auto\-Builder\)

### 1\. 项目核心目标 \(Core Objectives\)

__核心定位：__ 打造一个轻量级、高度定制化的本地开发者辅助工具（Developer Tool），运行于 VS Code 工作区内。

__效能目标：__ 将独立游戏开发初期（如原型搭建阶段）繁琐的数值策划、配置表编写和占位文本生成工作自动化，减少至少 80% 的手动数据录入与格式排版时间。

__质量目标：__ 确保 AI 生成的所有游戏配置文件（如 JSON/CSV）达到 __100% 的语法正确率__与__严格的结构一致性__，使其能够被 Unity、Godot 等主流游戏引擎直接读取，实现“零修改可用”。

### 2\. 项目需求 \(Project Requirements\)

本系统的需求围绕“自然语言理解 \-> 逻辑推理 \-> 本地文件生成”的闭环展开：

__REQ\-01 意图解析与上下文管理 \(Intent & Context Handling\)：__

系统需通过 Cline 接收开发者输入的非结构化自然语言指令（例如：“设计一个拥有三个阶段、带有冰冻技能的关卡 Boss”）。

系统需能在单次会话中维持上下文，确保生成的怪物数值与设定的游戏基础数值体系（如等级上限、伤害公式）不产生逻辑冲突。

__REQ\-02 结构化游戏数据生成 \(Structured Data Generation\)：__

通过调用 DeepSeek API，将推演出的游戏逻辑转化为预先定义好的数据格式（如 CharacterStats, ItemAttributes, DialogueTree）。

__强约束：__ 必须支持基于模式（Schema）的生成约束，拒绝任何带有 Markdown 标记或多余解释性文本的脏数据输出。

__REQ\-03 自动化本地 I/O 执行 \(Automated Local I/O\)：__

借助 Cline 的执行权限，系统需具备在 VS Code 当前工作区内自动创建标准目录树（如 \./Assets/Data/Enemies/）的能力。

系统需能自动将校验合格的 JSON 数据写入目标文件，并支持对已有配置文件的增量更新或安全覆盖。

__REQ\-04 容错与自愈机制 \(Fault Tolerance & Self\-Healing\)：__

当 DeepSeek 返回的格式损坏或缺少必填字段（如遗漏了 Boss 的 HP 属性）时，底层脚本必须能够自动捕获异常，并携带错误日志重新向 API 发起请求，且整个重试过程对用户（您）保持静默。

### 3\. 技术指标与约束 \(Technical Specifications\)

为了保证该本地工具的稳定性和轻量化，设定以下工程指标：

__运行环境与技术栈：__

__宿主环境：__ Visual Studio Code。

__执行代理：__ Cline 扩展（负责监听指令并执行本地终端命令）。

__脚本语言：__ Node\.js 或 Python（推荐使用 Node\.js，因为其处理 JSON 极其原生，且与 VS Code 生态契合度高；若您更熟悉 Python 的数据处理也可选择 Python）。

__核心大脑：__ DeepSeek Chat API。

__数据校验标准 \(Data Validation\)：__

采用严格的 Schema 校验库（Python 环境下使用 __Pydantic__，Node\.js 环境下使用 __Zod__）。

__指标：__ 配置文件格式错误拦截率需达到 100%；API 自动重试次数上限设定为 3 次。

__响应与执行性能 \(Performance\)：__

从发出指令到本地生成一个中等复杂度（约 50\-100 行 JSON）的游戏实体配置文件，端到端耗时应控制在 __5~10 秒__以内（取决于网络延迟）。

__工程解耦 \(Decoupling\)：__

Prompt 模板库、DeepSeek API 请求模块、本地文件读写模块必须在代码层面上完全分离，以便于后期随时调整游戏的数值规则，而不需要修改底层的网络请求代码。
